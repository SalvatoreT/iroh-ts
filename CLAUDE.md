## What This Is

TypeScript bindings for [iroh](https://iroh.computer), a peer-to-peer networking stack. Rust core compiled to WASM via `wasm-pack`, with thin TypeScript re-export layers for Node.js and browser environments.

## Commands

```bash
pnpm install                # install dependencies
pnpm build                  # full build: WASM (node + browser) then TypeScript
pnpm build:wasm             # rebuild WASM only (both targets)
pnpm build:ts               # rebuild TypeScript only (tsc)
pnpm test                   # run all tests (vitest)
pnpm test -- index.spec.ts  # run a single test file
```

### Build Prerequisites

Rust (stable), `wasm-pack`, `wasm32-unknown-unknown` target, pnpm. On macOS, the `ring` crate needs LLVM:

```bash
brew install llvm
export CC=/opt/homebrew/opt/llvm/bin/clang
export AR=/opt/homebrew/opt/llvm/bin/llvm-ar
```

## Architecture

### Layered build pipeline

1. **Rust WASM core** (`crate/src/`) â€” Rust modules (`endpoint.rs`, `addr.rs`, `blobs.rs`, `docs.rs`) use `wasm-bindgen` to expose classes to JS. Built twice by `wasm-pack`: once for Node.js (`crate/pkg/nodejs/`) and once for bundlers (`crate/pkg/bundler/`).

2. **TypeScript re-exports** (`ts/`) â€” Three files, no logic:
   - `node.ts` â†’ re-exports from `crate/pkg/nodejs/`
   - `browser.ts` â†’ re-exports from `crate/pkg/bundler/`
   - `index.ts` â†’ default re-export (Node.js)

   `package.json` `exports` map routes consumers to the right entry point based on environment.

3. **Compiled output** (`dist/`) â€” `tsc` compiles `ts/` here. Published alongside `crate/pkg/nodejs` and `crate/pkg/bundler`.

### Key types exposed

`Endpoint`, `EndpointAddr`, `Connection`, `BiStream`, `SendStream`, `RecvStream`, `BlobStore`, `DocEngine`, `Doc` â€” all generated by `wasm-bindgen` from the Rust side.

### QUIC stream behavior

`acceptBi()` won't resolve until the opener writes to its `SendStream`. Tests and examples must write before awaiting the accept side.

### WASM memory management

All WASM objects have `free()` and support `Symbol.dispose`. Call `free()` when done to avoid leaks.

## Testing

Tests are integration tests that create real iroh endpoints and connect them over relay servers. They require network access and have a 30-second timeout configured in `vitest.config.ts`.

## Commit Convention

All commit messages **must** use [Conventional Commits v1.0.0](https://www.conventionalcommits.org/en/v1.0.0/) format:

```
<type>(<optional scope>): <description>

[optional body]

[optional footer(s)]
```

### Rules

- **type** is required. Allowed types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`, `style`, `perf`, `build`.
- **scope** is optional and should be a noun describing the section of the codebase (e.g., `wasm`, `endpoint`, `blobs`, `ci`).
- **description** is required, lowercase, imperative mood, no period at the end.
- **body** is optional. Use it for additional context when the description alone is not sufficient. Separate from the description with a blank line.
- **footer(s)** are optional. Use `BREAKING CHANGE: <description>` in the footer to indicate a breaking API change (this correlates with a MAJOR version bump in SemVer). Other footers follow the `key: value` format (e.g., `Reviewed-by: Name`).
- Appending `!` after the type/scope (e.g., `feat!:` or `feat(endpoint)!:`) also indicates a breaking change and may be used with or without a `BREAKING CHANGE:` footer.
- `feat` correlates with a MINOR version bump; `fix` correlates with a PATCH version bump.

### Examples

```
feat(endpoint): add support for custom ALPN protocols
```

```
fix(blobs): prevent double-free on blob handle drop
```

```
docs: update CLAUDE.md with full conventional commits spec
```

```
feat(endpoint)!: change accept() to return Result instead of Option

BREAKING CHANGE: accept() now returns a Result, callers must handle the error variant.
```

```
chore: upgrade wasm-bindgen to 0.2.100
```
